Design systems have become essential infrastructure for product teams building across multiple platforms. A well-structured token system creates a single source of truth that propagates through every layer of the application.

Why Tokens Matter

Tokens are the atomic values of a design system. Colors, spacing, typography, shadows, and border radii are all defined as tokens that components consume. When a token changes, every component updates automatically.

Tokens bridge the gap between design intent and implementation reality. They ensure that what designers envision is precisely what engineers build.

With Tamagui, tokens are defined once in a shared configuration package and consumed across all components and themes. The compiler optimizes these at build time, extracting static styles for better runtime performance.

Platform Adaptation

iOS users expect centered navigation titles and subtle shadows, while Android users are accustomed to left-aligned titles and elevation-based depth. The design system handles these differences through platform-specific token overrides.

Typography follows the same adaptive pattern: SF Pro on iOS for that native Apple feel, Roboto on Android for Material consistency. Both are configured through the font token system with matching size scales.

Getting Started

The showcase app serves as living documentation for the full design system. Browse tokens, preview components, and explore templates to understand the available building blocks before integrating into your project.
